Notes:

- This matrix is scoped to current ecosystem repos: `pact-network-core-bun` (`core`), `pact-sdk` (`sdk`), and `pact-contracts` (`contracts`).
- Batch 34 sync is anchored to `core` commit `a6bdfb3` and `sdk` commit `811f4ff`, which extend adapter-health reporting, settlement connector idempotency/health/reset flows, compute runtime checkpoint/retry/cancel contracts, file-backed data-asset metadata, and SDK parity for those endpoints.
- `production-hardening` means the protocol surface is implemented and tested, but still relies on local/mock adapters or repo-local durability for part of the production path.

## Remaining production gaps

- Non-stablecoin settlement rails still need live provider-authenticated connectors for LLM billing, cloud credits, and API quotas; current connectors remain local/in-memory.
- Governance and rewards flows now have contract and SDK/runtime bridge coverage, but the runtime bridge still lacks live signer, RPC finality confirmation, and reorg/indexer handling.
- Compute, Data, and Dev now expose runtime and health contracts, but they still depend on local filesystem or in-memory adapters instead of managed lineage/proof stores, remote executors, and production integration backends.
- Durable state is now local-first (`file-backed`/SQLite), but production deployments still need managed database, queue, and observability backends plus runbook-level operational hardening.
- Appendix C remains open: there are no production zk circuits/provers yet, only schemas, validation primitives, and simulation/test coverage.
