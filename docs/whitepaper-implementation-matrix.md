## Remaining production gaps

- Non-stablecoin settlement rails now ship merged JSON+env provider-profile loading, credential-alias normalization, authenticated HTTP transport contracts, request/response digest validation, idempotency headers, retries, circuit-breaker state, and idempotent reconciliation hooks, but production rollout still needs live provider deployments, secret rotation, and audited rate/source integrations.
- Governance and rewards flows now ship signer abstractions, RPC-backed finality providers, validated contract addresses, pending-nonce synchronization, reorg-aware indexer hooks, and env-wired container bridge configuration, but production deployments still need persistent indexer storage, signer custody/HSM integration, and multi-network operations hardening.
- Compute, Data, and Dev now ship env-configured managed-backend inventory loading, JSON+env backend profile merging, managed-backend and dev-integration health contracts, lifecycle mirroring for data publications/access policies/listings/purchases/integrity proofs, and remote queue/store/observability skeleton coverage, but they still depend on skeleton adapters and local filesystem/in-memory defaults rather than real managed backends.
- Durable state remains local-first (`file-backed`/SQLite) for most runtime stores, so production deployments still need managed database, queue, and observability backends plus runbook-level operational hardening.
- Appendix C now ships manifest versioning, manifest-catalog health classification, bridge runtime metadata, env/profile-backed remote prover configuration, a real remote HTTP prover adapter with timeout/auth/request- and response-digest validation for prove/verify plus remote artifact fetches, degraded skeleton fallback, binary-safe artifact integrity validation, SDK runtime/manifest/adapter-health accessors, and verification receipt traceability, but production rollout still needs externally operated prover services, secret rotation, and circuit artifact distribution hardening.
